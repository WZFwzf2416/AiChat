# Agent 演进路线图

## 目标
把当前的 AI Chat 项目逐步演进成一个可观察、可调试、可扩展的 Agent Runtime，并且保持对前端开发者友好。

## 阶段一：单轮 Agent 基础设施稳定化
- 清理用户可见文案和编码问题，统一使用 UTF-8 中文文案。
- 明确消息可见性语义：`visible`、`debug`、`internal`。
- 为每一轮对话建立统一的 `turnId`，让消息、步骤、诊断数据能按轮次关联。
- 让工具错误进入标准消息链路和 Agent 轨迹，而不是直接中断整轮。
- 在 Inspector 中展示最近轮次、Agent 轨迹和隐藏调试消息。

## 阶段二：单 Agent 循环执行
- 将当前 `plan -> tool -> answer` 升级为可循环的 `think -> act -> observe -> think -> finalize`。
- 抽离 `runtime`、`planner`、`executor`、`streaming` 模块，避免 `providers.ts` 继续膨胀。
- 支持最大步数、提前终止条件和循环级错误恢复。
- 把每一步思考、工具调用、观察结果都结构化记录为 step。

## 阶段三：工具系统平台化
- 为工具增加分类、权限和运行策略。
- 统一工具输出协议：`summary`、`raw`、`sources`、`display`。
- 支持工具超时、重试、失败分级和埋点。
- 让前端基于结构化输出渲染卡片，而不是只显示普通文本块。

## 阶段四：知识库升级为长期记忆
- 区分会话记忆和长期知识。
- 支持从对话中提炼知识条目，并提供人工确认入口。
- 为知识条目增加来源会话、来源消息、确认状态、命中次数。
- 让知识库从“附属工具”变成 Agent 记忆模块。

## 阶段五：前端升级为 Agent 工作台
- 在消息区展示阶段感和执行反馈，而不只是最终回复。
- 将 Inspector 升级为完整的运行面板，展示上下文裁剪、工具输入输出、步骤和引用来源。
- 对天气、时间、知识库结果做卡片化展示。
- 强化教学感，让用户看清“模型判断”和“工具结果”分别是什么。

## 阶段六：多 Agent 协作
- 在单 Agent 运行稳定后，再引入多 Agent。
- 先从职责型拆分开始，例如 `Research`、`Writer`、`Memory`。
- 或按流程型拆分，例如 `Planner`、`Executor`、`Reviewer`、`Finalizer`。
- 多 Agent 必须建立在统一 step 模型、工具协议和调试视图之上。

## 近期执行顺序
1. 完成阶段一的基础设施和前端可视化。
2. 拆分 `providers.ts`，为阶段二的循环执行做准备。
3. 规范工具输出协议，减少未来重构成本。
4. 将知识库和 Inspector 面板逐步升级为真正的记忆与观测系统。
